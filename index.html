A Lightless Labs research project

What AI coding agents actually do, from the logs up.

When you use a coding agent — Claude Code, Codex, that sort of thing — every turn gets written to a transcript: your messages, the model's replies, its internal thinking, the tool calls it makes. Most people skim past those transcripts. We read them, in bulk, and count what's in there. Not what the agent says it did. Not the one demo that worked on a Tuesday.

The premise

The transcripts are where the behavior lives.

An agent run leaves a long, messy record: every message, every file touched, every tool invoked, every time the model second-guesses itself in its internal reasoning. That's the evidence — ugly, repetitive, implementation-detail-heavy — and it's where you have to go if you want to figure out what these systems are actually doing, rather than what they or their vendors say they're doing.

Why bother

Otherwise you get opinions with nicer typography.

Most claims about what AI coding agents do well or badly come from somebody's recollection of a handful of sessions they remember well. Those aren't nothing, but they're not much either. The boring version — measure the same things across thousands of sessions, see what survives, retract what doesn't — gives you a different kind of knowing. Half of this project is the tool that does the measuring. The other half is writing up what it turned up, including the bits that didn't hold.

Questions we've been chasing

Going in, we didn't know which of these would turn out to have clean answers. Most didn't, on the first pass. A few did. The rest are still being argued with.

On internal reasoning

Does the model's thinking show up in what users see?

Reasoning models produce enormous amounts of internal thinking compared to what they actually reply — roughly 700 tokens of private reasoning for every token of user-facing text. We wanted to know whether hedges and worries inside the thinking ("this could fail," "I'm not sure") leak into the reply, or whether they mostly stay hidden. First pass says: mostly stay hidden.

On good vs bad runs

What distinguishes a session that worked from one that didn't?

First we had to pick a usable definition of "worked." The one we settled on: the user didn't feel the need to step in and correct the agent. Then we went looking for behaviors that show up disproportionately in one group and not the other. The clearest signal so far is self-correction tied to evidence-gathering.

On drifting runs

Can you tell a session is going sideways before the user steps in?

Agents seem to fall into a recognizable "gone quiet, not producing much" state before users intervene. Whether that's a useful early-warning signal depends on how often it shows up without a correction following. First pass says: maybe, and the effect is smaller than we initially claimed.

Across tools and versions

Do different agents behave differently?

The parser already speaks four transcript formats (Claude Code, Codex, OpenCode, Gemini), and the question naturally generalises: same CLI across model versions, same task across different agents, same agent before and after a rollout. The honest status is that the corpus isn't yet balanced enough across those axes to answer cleanly — most of it is still Claude Code. It's the question that gets more interesting as the corpus widens, and the one most likely to surface a rollout or vendor change the moment it lands.

Install middens

middens is the Rust command-line tool that does the digging. Point it at a folder of session transcripts, and it parses them, classifies the messages, runs a battery of analyses, and writes out results you can read without squinting at raw JSON.

From GitHub Releases

Download the tarball for your platform.

Grab the latest tarball from GitHub Releases, unpack it, run ./middens --help. Homebrew is coming — once the release plumbing stops behaving like release plumbing. 

# macOS (Apple Silicon) or Linux
curl -LO https://github.com/Lightless-Labs/third-thoughts/releases/latest/download/middens-<version>-<target>.tar.gz
tar -xzf middens-*.tar.gz
cd middens-*/
./middens run ./corpus/ --all
