A Lightless Labs research project

What AI coding agents actually do, from the logs up.

When you use a coding agent — Claude Code, Codex, that sort of thing — every turn gets written to a transcript: your messages, the model's replies, its internal thinking, the tool calls it makes. Most people skim past those transcripts. We read them, in bulk, and count what's in there. Not what the agent says it did. Not the one demo that worked on a Tuesday.

The premise

The transcripts are where the behavior lives.

An agent run leaves a long, messy record: every message, every file it touched, every tool it invoked, every time it second-guessed itself in its internal reasoning. That's the evidence — ugly, repetitive, implementation-detail-heavy — and it's where you have to go if you want vague hunches about agent behavior to turn into numbers that can survive a second look.

Why it matters

Otherwise you get opinions with nicer typography.

A corpus of logs can show you which working patterns correlate with sessions that went well, when a session is drifting toward trouble before it fully falls over, and whether a new tool version quietly made things worse. Half of this project is a command-line tool. The other half is writing up what the tool turned up.

Catch behavior changes

See when a rollout made things worse.

A new CLI version or a new model can shift agent behavior in ways nobody notices until the bug tracker catches fire a week later. Run the same analyses on a before-and-after corpus and see what moved — the logs are already versioned for you.

Good vs bad sessions

Tell a productive run from a drifting one.

Sessions where users don't have to step in have a recognizable shape: the agent keeps checking its own work while it's looking things up. Sessions that need heavy intervention tend to have a lot of planning language and very little self-correction. Those two shapes are empirically different and we can count them.

Flag trouble early

Catch sessions drifting toward correction.

Before a user jumps in to correct an agent, the agent tends to go quiet and stop producing — a recognizable "I'm stuck but still typing" state. We can pick it out of the logs. The direction looks robust; the size of the effect is still being pinned down.

Install middens

middens is the Rust command-line tool that does the digging. Point it at a folder of session transcripts, and it parses them, classifies the messages, runs a battery of analyses, and writes out results you can read without squinting at raw JSON.

From GitHub Releases

Download the tarball for your platform.

Grab the latest tarball from GitHub Releases, unpack it, run ./middens --help. Homebrew is coming — once the release plumbing stops behaving like release plumbing. 

# macOS (Apple Silicon) or Linux
curl -LO https://github.com/Lightless-Labs/third-thoughts/releases/latest/download/middens-<version>-<target>.tar.gz
tar -xzf middens-*.tar.gz
cd middens-*/
./middens run ./corpus/ --all
